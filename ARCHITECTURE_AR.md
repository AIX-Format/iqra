## 🔧 المكونات الأساسية الثلاثة

### 1️⃣ Soul Engine — المحرك الروحاني

**الملف:** `lib/iqra/soul_engine.ts`

**الوظيفة:** إدارة دورات التطور الثلاثية

```typescript
// دورة 3-6-9 (من تسلا)
Pulse 3  → Reflection (التفكر)    — تحليل الذاكرة الحديثة
Pulse 6  → Evolution (التطور)     — تحديث الأنماط
Pulse 9  → Wisdom (الحكمة)        — استخراج القوانين الكبرى
```

**المنطق:**
- كل مهمة تزيد عداد `cycleCounter`
- عند كل 3 نبضات → تفكر (تحليل المكافآت والأخطاء)
- عند كل 6 نبضات → تطور (تحديث ذاكرة الأنماط)
- عند كل 9 نبضات → حكمة (استخراج قوانين جديدة)

**الفائدة:**
- النظام يتعلم من تجاربه
- الأنماط تتحسن مع الوقت
- الأخطاء تُصحح ذاتياً

---

### 2️⃣ Damir Conscience — الضمير النانوي

**الملف:** `lib/iqra/damir_conscience.ts`

**الأساس النظري:** Graded Linear Logic (جيرار، 1987)

```
المنطق الكلاسيكي:  المعلومة تُنسخ بحرية
المنطق الخطي:      كل مورد يُستهلك مرة واحدة فقط
```

**الفحوصات الثلاثة:**

```
1. فحص النية (Intention Check)
   ↓
   هل النية تحتوي على كلمات محرمة؟
   (كذب، تضليل، احتيال، إلخ)
   
2. فحص الموارد (Resource Check)
   ↓
   هل المورد موجود؟ هل مستهلك بالفعل؟
   هل مصدره حقيقي أم مزيف؟
   
3. فحص الثقة (Confidence Score)
   ↓
   درجة الثقة = 0.5 إلى 1.0
   (تعتمد على جودة الموارد)
```

**السرعة:** < 5ms (بدون LLM، بدون شبكة)

**الفائدة:**
- حماية من الأفعال المحرمة
- منع استخدام الموارد المزيفة
- تسجيل كل قرار في TrustChain

### 3️⃣ Heartbeat System — نبض الحياة

**الملف:** `lib/iqra/heartbeat.ts`

**دورات النبض:**

| الفترة | الحدث | الوظيفة |
|--------|-------|---------|
| كل 9 ثانية | MICRO_PULSE | تحديث عداد النبض + Pulse369 |
| كل 27 ثانية | WARM_PULSE | فحص Redis + Qdrant + تحديث Curiosity |
| كل 81 ثانية | DEEP_PULSE | تحليل قرآني + تقرير |
| كل 3 دقائق | HEALTH_CHECK | فحص شامل لكل الأنظمة |
| كل 9 دقائق | DISCOVERY | دورة اكتشاف قرآني |
| كل 40 دقيقة | TAZKIYAH | تطهير الذاكرة |

**الفائدة:**
- النظام يراقب نفسه بنفسه
- الأخطاء تُكتشف فوراً
- التقارير تُرسل للـ Telegram

## 🧠 طبقات الذاكرة السبع

```
Layer 1: Working Memory (RAM — 49 مدخل)
         ↓ سريع جداً (< 1ms)
         ↓ يُمسح كل 40 دقيقة

Layer 2: Episodic Memory (SQLite)
         ↓ متوسط (10-50ms)
         ↓ يحفظ التجارب

Layer 3: Semantic Memory (Qdrant)
         ↓ بطيء (100-500ms)
         ↓ يحفظ المعاني والأنماط

Layer 4: Procedural Memory (SkillBank)
         ↓ يحفظ المهارات والخوارزميات

Layer 5: Topological Memory (Persistent Homology)
         ↓ يحفظ الأشكال والهياكل

Layer 6: Quantum Topology (Qdrant)
         ↓ يحفظ الأنماط العابرة

Layer 7: Constitutional Memory (DASTŪR.md)
         ↓ ثابت، لا يتغير
         ↓ القوانين الأساسية
```

**الترقية (Promotion):**
```
Hot (Layer 1) → Warm (Layer 2) → Cold (Layer 3)
     ↑                                    ↓
     └────────── Pulse369.tick() ────────┘
```
