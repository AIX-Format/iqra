### 1. contracts.ts — الفجوات الرئيسية

**الحالة الحالية:**
- ✅ تعريف الأنواع الأساسية (WorkerRole, SourceTag, SourceAttestation)
- ✅ تعريف الواجهات (ContextSnapshot, CommandLog, WorkerReport, MissionHandoff)

- ❌ **دالة validateWorkerAction() مفقودة**
- ❌ **دالة validateSourceAttestations() مفقودة**
- ❌ **دالة validateNoMock() مفقودة**

**الفجوات:**

#### الفجوة 1: validateWorkerAction() — فرض القيود
**الوصف:** دالة للتحقق من أن الوكيل لا يتجاوز قيوده
**المتطلب:** Req-5 من requirements.md
**التفاصيل:**
```typescript
// مفقود:
export function validateWorkerAction(
  worker: WorkerRole,
  action: WorkerAction
): { valid: boolean; reason?: string }
```

**السبب:** لا توجد آلية لفرض القيود الهيكلية (مثل: Validator لا يمكنه تعديل الكود)

**التأثير:** 
- لا يمكن منع Validator من تعديل implementation logic
- لا يمكن منع Reporter من كتابة الكود
- لا يمكن منع Builder من تجاوز التحقق

---

#### الفجوة 2: validateSourceAttestations() — شهادة المصدر
**الوصف:** دالة للتحقق من أن كل ادعاء له مصدر
**المتطلب:** Req-6 من requirements.md
**التفاصيل:**
```typescript
// مفقود:
export function validateSourceAttestations(
  report: WorkerReport
): { valid: boolean; missing: string[] }
```

**السبب:** لا توجد آلية للتحقق من أن كل ادعاء في التقرير موثق بمصدر

**التأثير:**
- يمكن للوكلاء تقديم ادعاءات بدون مصادر
- لا يمكن التحقق من صحة المعلومات
- ينتهك القاعدة الدستورية #3: "كل مصدر معلومة يُوسَم"

---

#### الفجوة 3: validateNoMock() — فرض عدم وجود Mock
**الوصف:** دالة للتحقق من أن لا mock في الإنتاج
**المتطلب:** Req-7 من requirements.md
**التفاصيل:**
```typescript
// مفقود:
export function validateNoMock(
  report: WorkerReport,
  env: 'production' | 'development'
): { valid: boolean; reason?: string }
```

**السبب:** لا توجد آلية لفرض عدم وجود mock في بيئة الإنتاج

**التأثير:**
- يمكن استخدام simulated providers في الإنتاج
- ينتهك القاعدة الدستورية #2: "لا mock ولا simulated provider في بيئة الإنتاج"
- يؤثر على موثوقية النتائج
